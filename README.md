# Axiozen

Marketing + product catalogue site for **Axiozen** — _Solutions That Secure, Automate & Empower_.
Built with Next.js (App Router) and TypeScript. Covers website design, hosting, app development,
biometric devices, CCTV, gate automation, security solutions, and attendance & payroll software.

## Getting Started

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

The admin panel lives at `/admin` (default dev credentials: `admin` / `axiozen@admin`).

## Configuration

- **Branding & contact details:** `lib/config.ts` (company name, email, phone, address, socials).
  These currently hold placeholders — update them with real Axiozen details.
- **Colors:** CSS variables in `public/css/style.css` (`:root`) plus a few accents in `app/globals.css`.
- **Logo:** `public/images/logo.png`. Favicon is generated by `app/icon.tsx`.

## Data & environment

- **Local dev:** SQLite at `data/axiozen.db` (schema auto-created on first run).
- **Production:** set `DATABASE_URL` to use Neon Postgres instead of SQLite.

Optional env vars: `RESEND_API_KEY`, `MAIL_FROM`, `MAIL_TO` (contact emails),
`BLOB_READ_WRITE_TOKEN` (Vercel Blob uploads), `ADMIN_USERNAME`, `ADMIN_PASSWORD`,
`SESSION_SECRET`, `GOOGLE_PLACES_API_KEY`, `GOOGLE_PLACE_ID`, `YOUTUBE_CHANNEL_ID`,
`NEXT_PUBLIC` analytics IDs (`gtmId` / `gaMeasurementId` in `lib/config.ts`).

## Deploy on Vercel

```bash
vercel --prod
```
